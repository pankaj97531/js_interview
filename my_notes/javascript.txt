A : There are three type of error in javascript.

	1.Syntax error(Parsing error).
		Occur at compile time for traditional programming language. But in javascript it occur at interpret time.
		Syntax error affect(reflect) total program. 
		example : function test(){
		
	2. Runtime Error :
	Runtime error are also called exception occur during exection(after compilation/execution);
	This error is handled by try catch finally block.
	Finally block will always get executed.
		example  : 
		Uncaught SyntaxError: Identifier 'testname' has already been declared
		
	3. 	Logical error : 
	Error occur due to logical problem.

Q : difference between cookies , localstorage and session storage
Q : 
Q : LocalStorage
Q : 
Q : Stores data with no expiration date, and gets cleared only through JavaScript, or clearing the Browser cache / Locally Stored Data
Q : 
Q : Storage limit is the maximum amongst the three
Q : 
Q : localStorage.setItem('myCat', 'Tom');
Q : 
Q : var cat = localStorage.getItem('myCat');
Q : 
Q : localStorage.removeItem('myCat');
Q : 
Q : localStorage.clear();
Q : 
Q : SessionStorage
Q : 
Q : The sessionStorage object stores data only for a session, meaning that the data is stored until the browser (or tab) is closed.
Q : 
Q : Data is never transferred to the server.
Q : 
Q : Storage limit is larger than a cookie (at least 5MB).
Q : 
Q : sessionStorage.setItem('key', 'value');
Q : 
Q : let data = sessionStorage.getItem('key');
Q : 
Q : sessionStorage.removeItem('key');
Q : 
Q : sessionStorage.clear();
Q : 
Q : Cookie
Q : 
Q : Stores data that has to be sent back to the server with subsequent requests. Its expiration varies based on the type and the expiration duration can be set from either server-side or client-side (normally from server-side).
Q : 
Q : Cookies are primarily for server-side reading (can also be read on client-side), localStorage and sessionStorage can only be read on client-side.
Q : 
Q : Size must be less than 4KB.
Q : 
Q : Cookies can be made secure by setting the httpOnly flag as true for that cookie. This prevents client-side access to that cookie
Q : 
Q : In order to delete a cookie set the expires date to something in the past.
